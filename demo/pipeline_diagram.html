🔍 What Happens Inside Each Forward Pass
This is what the model does with your image — identical for both Run A and Run B
STAGE 1

🖼️ Image Tokenizer (DINOv2 ViT)

Encodes the 2D image into 1024 visual tokens, each a 1024-dim vector.
These tokens capture visual features: edges, textures, object shape from the photo.

Image (512×512×3) → DINOv2 → tokens (B, 1024, 1024)
STAGE 2

🎯 Scene Tokenizer (Learnable Embeddings)

Creates 1024 learnable query tokens — think of these as "empty 3D slots" that will learn the shape.

Learnable embeddings → query tokens (B, 1024, 1024)
STAGE 3 — THE CORE

🧠 Transformer Backbone — 32 Attention Layers ⏱ TIMED

This is where the magic happens. Scene tokens learn the 3D structure by attending to image tokens and each other.

Pattern repeats 16 times: Cross-Attention → Self-Attention
Layer 1 CROSS-ATTENTION PyTorch SDPA
Layer 2 ⭐ SELF-ATTENTION 🔥 OUR TRITON KERNEL
Layer 3 CROSS-ATTENTION PyTorch SDPA
Layer 4 ⭐ SELF-ATTENTION 🔥 OUR TRITON KERNEL
... × 16 pairs total = 32 layers
Layer 31 CROSS-ATTENTION PyTorch SDPA
Layer 32 ⭐ SELF-ATTENTION 🔥 OUR TRITON KERNEL
Why only self-attention? Cross-attention has different sequence lengths (Q=1024 scene tokens, K/V=1024 image tokens from a different source). Our kernel requires Q and K/V to have the same sequence length. So cross-attention falls back to PyTorch — graceful degradation!

STAGE 4

📐 Post-Processor → Triplane Features

Converts transformer output tokens into triplane features — three axis-aligned 2D feature maps (XY, XZ, YZ planes) that encode the 3D shape.

tokens (B, 1024, 1024) → triplanes (B, C, 64, 64) × 3 planes

STEP 5

🔷 Mesh Extraction (Marching Cubes) NOT TIMED — same for both

This step is identical for both Run A and Run B. It uses the triplane features (which are nearly identical since cosine similarity ≈ 1.0).

1. Sample 3D Grid
Query 256³ = 16.7M points in 3D space
2. Query Triplane Decoder
For each point: project onto 3 planes → decode density
3. Marching Cubes
Extract surface where density > 25.0 → vertices + faces
4. Color Extraction
Query triplane at each vertex for RGB → colored mesh
~2000 ms (NOT part of the speed comparison)
